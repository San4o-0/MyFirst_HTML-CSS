1. Я розпочав лабораторну роботу зі створення нового проєкту Vue 3 з нуля за допомогою команди npm create vue@latest.
Під час ініціалізації я обрав налаштування з TypeScript та Vue Router, щоб одразу мати можливість працювати з типізацією
та маршрутизацією. Після цього я ознайомився зі структурою створеного проєкту та ключовими файлами: main.ts, де
відбувається підключення Vue, маршрутизатора і глобальних стилів; router/index.ts, де налаштовується маршрутизатор
для переходу між сторінками; а також App.vue, який є головним компонентом програми і точкою входу для всіх інших компонентів.
Далі я додав нову сторінку /contact, створивши файл ContactView.vue, і зареєстрував її у маршрутизаторі, щоб можна було
переходити на цю сторінку за допомогою Vue Router. Після цього я створив компонент UserCard.vue, у якому чітко вказав
типи пропсів за допомогою TypeScript, що дозволяє уникати помилок при передачі даних і робить код більш надійним та зрозумілим.
2. У процесі виконання лабораторної роботи я розкоментував рядок з помилкою типізації в ContactView.vue, щоб перевірити,
як TypeScript реагує на невідповідність типів. При спробі присвоїти змінній типу boolean значення number з’явилася
помилка, яка була відразу помічена як у VS Code, так і в терміналі під час запуску команди npm run dev. Коментар у
коді виглядав так: // const isActive: boolean = 123 // ❌ Type 'number' is not assignable to type 'boolean', а текст
помилки у консолі показував TS2322: Type 'number' is not assignable to type 'boolean'. --> src/views/ContactView.vue:17:7.
Це наочно показало, як TypeScript допомагає ловити помилки ще на етапі розробки.
3. Висновки, які я зробив після виконання цієї лабораторної роботи, полягають у тому, що використання TypeScript у поєднанні
з Vue Router значно підвищує надійність і зрозумілість проєкту. TypeScript дозволяє контролювати типи даних і запобігати
помилкам, які могли б виникнути під час виконання коду, а Vue Router дає змогу легко організовувати багатосторінкову
SPA-структуру та швидко додавати нові маршрути і сторінки. Поєднання цих технологій робить проєкт більш масштабованим та
зручним для підтримки.
4. Після виконання всіх завдань я зробив коміт у свій репозиторій, додавши файл Lab.txt та зміни у компонентах, і відправив
їх на сервер за допомогою інтерфейсу WebStorm.
