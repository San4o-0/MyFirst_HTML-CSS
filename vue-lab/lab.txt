1. Етапи виконання та виправлення помилок
Проєкт ініційовано командою npm create vue@latest vue-lab із вибором TypeScript та Vue Router

Початкову помилку ENOENT виправлено переходом до коректної директорії проєкту vue-lab

Виправлено помилки компілятора шляхом відновлення обов'язкових блоків <script> та <template> у порожніх файлах HomeView.vue та AboutView.vue

З кореневого компонента App.vue видалено посилання на відсутній файл logo.svg

Додано новий маршрут /contact до конфігурації Vue Router, використовуючи динамічний імпорт (ліниве завантаження)

Створено компонент сторінки ContactView.vue та додано відповідне навігаційне посилання в App.vue

2. Демонстрація типізації
Створено компонент UserCard.vue, де пропси username (string) та isActive (boolean) були чітко типізовані за допомогою defineProps

Компонент імпортовано та використано на сторінці контактів

Під час спроби передати числове значення замість булевого для пропсу isActive система видала помилку типізації: Type 'number' is not assignable to type 'boolean'

Цей текст помилки, виявлений на етапі розробки, було додано до звіту як коментар

3. Висновки про Vue 3, TS та Vue Router
Використання TypeScript у проєкті забезпечує високу надійність коду, дозволяючи виявляти помилки типів даних, як-от некоректна передача пропсів, ще на етапі написання

TypeScript покращує інструментарій розробника, надаючи інтелектуальне автодоповнення та роблячи код більш зрозумілим для команди

Vue Router дозволяє створити повноцінний Single Page Application (SPA) із плавними переходами між сторінками

Застосування лінивого завантаження для маршрутів є критичним для оптимізації продуктивності, оскільки скорочує час початкового завантаження застосунку, відкладаючи завантаження коду до моменту, коли він справді необхідний користувачеві
